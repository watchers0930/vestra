# 개인정보 영향평가 (PIA)

> VESTRA AI 자산관리 플랫폼 — 개인정보보호법 제33조 기반

## 1. 평가 개요

| 항목 | 내용 |
|------|------|
| 시스템명 | VESTRA AI 부동산 자산관리 플랫폼 |
| 평가일 | 2026-03-11 |
| 평가자 | 개발팀 |
| 근거 법률 | 개인정보보호법 제33조, 동법 시행령 제35조 |

## 2. 수집 개인정보 항목

| 항목 | 수집 목적 | 보유 기간 | 암호화 |
|------|-----------|-----------|--------|
| 이메일 | 계정 식별, 로그인 | 탈퇴 시까지 | DB 평문 (OAuth 위임) |
| 이름 | 표시용 | 탈퇴 시까지 | DB 평문 |
| 프로필 이미지 | 표시용 | 탈퇴 시까지 | URL 저장 |
| 사업자등록번호 | 기업/부동산 등급 인증 | 탈퇴 시까지 | AES-256-GCM |
| 주소 (분석용) | 부동산 분석 요청 | 탈퇴 시까지 | AES-256-GCM |
| IP 주소 | 감사 로그, Rate Limit | 1년 | DB 평문 |
| User-Agent | 감사 로그 | 1년 | DB 평문 |

## 3. 개인정보 처리 흐름

```
[사용자] → OAuth 로그인 → [NextAuth] → [Neon PostgreSQL]
                                              ↓
[사용자] → 분석 요청 (주소) → [API Route] → [DB 암호화 저장]
                                              ↓
[관리자] → 대시보드 → [복호화 후 표시] → [마스킹 적용]
```

## 4. 위험 분석 및 대응

| 위험 요소 | 위험도 | 대응 조치 |
|-----------|--------|-----------|
| DB 유출 시 PII 노출 | 높음 | AES-256-GCM 암호화 (Prisma 미들웨어 자동) |
| XSS로 localStorage 탈취 | 중간 | Base64 인코딩 + sanitize.ts 다층 방어 |
| 세션 탈취 | 중간 | JWT + HttpOnly 쿠키 + SameSite=Lax |
| 관리자 계정 탈취 | 높음 | bcrypt(12) + 8자 이상 3종 조합 비밀번호 정책 |
| CSRF 공격 | 중간 | Origin/Referer 검증 + SameSite 쿠키 |
| 무차별 대입 | 중간 | DB 기반 Rate Limit (30req/60s + 일일 한도) |

## 5. 정보주체 권리 보장

| 권리 | 이행 방법 |
|------|-----------|
| 열람권 | /profile 페이지에서 본인 정보 확인 |
| 정정권 | 프로필 수정 기능 |
| 삭제권 | 회원 탈퇴 시 Cascade 삭제 (DB onDelete: Cascade) |
| 처리정지권 | 관리자에게 요청 → 계정 비활성화 |
