> **Summary**: 전세 분석 시 HUG·HF·SGI 3개 기관의 전세보증보험 가입 가능성을 자동 판단하여 제공

> **Date**: 2026-03-23
> **Status**: Draft

## 1. Overview

### 1.1 Purpose

전세 계약 정보를 입력하면 HUG(주택도시보증공사), HF(한국주택금융공사), SGI(서울보증) 3개 기관의 전세보증보험 가입 가능 여부를 자동으로 판단하여, 임차인이 어떤 보증보험에 가입할 수 있는지 한눈에 확인할 수 있도록 한다.

### 1.2 Background

- **내집스캔** 등 경쟁 서비스가 보증보험 가입 가능성 판단을 제공 중
- VESTRA의 전세 안전 분석(`/jeonse/analysis`)에는 전세권 설정 필요성·위험도 분석은 있으나 **보증보험 가입 가능성 판단이 없음**
- 각 기관의 가입 조건(보증금 한도, 담보인정비율, 전세가율 등)이 공개 정보이므로 외부 API 제휴 없이 규칙 기반으로 구현 가능
- 전세 임차인이 가장 궁금해하는 실질 정보 → 사용자 가치 높음

### 1.3 Related Documents

- 기존 전세 분석: `app/(app)/jeonse/analysis/page.tsx`
- 전세보호 허브: `app/(app)/jeonse/page.tsx`
- 참고: [2026 전세보증보험 가입조건 HUG·HF·SGI 비교](https://info.yiwi.co.kr/전세보증보험-가입조건/)

## 2. Scope

### 2.1 In Scope

- [x] HUG·HF·SGI 3개 기관 가입 조건 규칙 엔진 구현
- [x] 기존 전세 분석 결과에 보증보험 가입 가능성 섹션 추가

### 2.2 Out of Scope

- 보증보험 실제 신청/접수 (외부 사이트 연결만 제공)
- 임대인 동의 기반 신용조회 (SCI 제휴 필요)
- 전세대출 가심사 (금융기관 API 필요)

## 3. Requirements

### 3.1 Functional Requirements

| ID | Requirement | Priority | Status |
|----|-------------|----------|--------|
| FR-01 | 입력 정보(보증금, 주택유형, 주택가격, 선순위채권)로 3개 기관 가입 가능성 판단 | High | Pending |
| FR-02 | 기관별 판정 결과(가능/조건부/불가) + 상세 사유 표시 | High | Pending |
| FR-03 | 예상 보증료 계산 (보증금 × 보증료율 × 계약기간/365) | High | Pending |
| FR-04 | 가입 불가 사유별 해결 방안 안내 (예: 선순위채권 감축 필요) | Medium | Pending |
| FR-05 | 기존 전세 분석 API 응답에 보증보험 판단 결과 통합 | High | Pending |
| FR-06 | 보증보험 상세 조건 툴팁/모달로 각 기관 기준 안내 | Low | Pending |

### 3.2 Non-Functional Requirements

| Category | Criteria | Measurement Method |
|----------|----------|-------------------|
| Performance | 규칙 엔진 판단 < 50ms (서버사이드 계산) | 클라이언트 단 계산, API 호출 불필요 |
| Accuracy | 공식 기관 기준 대비 95% 이상 일치 | 공식 사이트 조건 교차 검증 |
| Maintenance | 기관별 조건 변경 시 config 파일만 수정 | JSON 기반 규칙 설정 |

## 4. 보증보험 가입 조건 (규칙 엔진 기준)

### 4.1 HUG (주택도시보증공사)

| 항목 | 조건 |
|------|------|
| 보증금 한도 | 수도권 7억 원 / 비수도권 5억 원 |
| 주택가격 | 12억 원 이하 |
| 담보인정비율 | (선순위채권 + 보증금) ≤ 주택가격 × 90% |
| 전세가율 | 공시가격의 126% 이내 |
| 계약기간 | 1년 이상 |
| 가입시기 | 계약기간 1/2 경과 전 |
| 주택유형 | 아파트, 연립·다세대, 단독·다가구, 주거용 오피스텔 |
| 기타 | 권리침해(압류·경매) 없을 것, 위반건축물 아닐 것 |
| 보증료율 | 연 0.115% ~ 0.154% (주택유형별 차등) |

### 4.2 HF (한국주택금융공사)

| 항목 | 조건 |
|------|------|
| 보증금 한도 | 은행별·상품별 상이 (일반적 수도권 5억) |
| 담보인정비율 | 90% 이내 (차등 적용) |
| 가입조건 | 전세자금대출 연계 필수 |
| 계약기간 | 1년 이상 |
| 주택유형 | 아파트, 연립·다세대, 단독·다가구, 주거용 오피스텔 |
| 보증료율 | 연 0.05% ~ 0.15% (대출 상품별 차등) |

### 4.3 SGI (서울보증)

| 항목 | 조건 |
|------|------|
| 보증금 한도 | 제한 없음 (고액 전세 가능) |
| 담보인정비율 | 약 90% |
| 계약기간 | 1년 이상 |
| 가입시기 | 계약기간 1/2 경과 전 |
| 주택유형 | 아파트, 연립·다세대, 단독·다가구, 주거용 오피스텔 |
| 보증료율 | 연 0.128% ~ 0.211% (주택유형·보증금별 차등) |

### 4.4 판단 로직 흐름

```
입력: 보증금, 주택유형, 주택가격(시세), 선순위채권액, 계약기간, 지역(수도권 여부)
  │
  ├─ 공통 검증
  │   ├─ 계약기간 ≥ 1년?
  │   ├─ 계약기간 1/2 미경과?
  │   └─ 주택유형 지원 여부?
  │
  ├─ HUG 판단
  │   ├─ 보증금 ≤ 한도 (수도권 7억/비수도권 5억)?
  │   ├─ 주택가격 ≤ 12억?
  │   └─ (선순위채권 + 보증금) ≤ 주택가격 × 90%?
  │
  ├─ HF 판단
  │   ├─ 전세대출 연계 여부? (사용자 입력)
  │   └─ (선순위채권 + 보증금) ≤ 주택가격 × 90%?
  │
  └─ SGI 판단
      └─ (선순위채권 + 보증금) ≤ 주택가격 × 90%?
```

## 7. Risks and Mitigation

| Risk | Impact | Likelihood | Mitigation |
|------|--------|------------|------------|
| 기관별 조건 변경 (연 1~2회) | Medium | High | JSON config 분리, 업데이트 용이하게 설계 |
| 주택가격 정확도 (사용자 입력 의존) | Medium | Medium | 시세 참고 안내 + 공시가격 API 연동 고려 |
| 실제 가입 시 추가 심사로 결과 상이 | Low | High | "참고용" 면책 문구 필수 표시 |

## 8. Architecture Considerations

### 8.1 Project Level

- **Dynamic** (기존 VESTRA 레벨 유지)

### 8.2 Key Decisions

| Decision | Selected | Rationale |
|----------|----------|-----------|
| 판단 엔진 위치 | 클라이언트 사이드 (lib/) | 외부 API 불필요, 순수 규칙 기반 계산 |
| 규칙 저장 형태 | TypeScript 상수 (lib/guarantee-rules.ts) | 타입 안전성 + 유지보수 용이 |
| UI 위치 | 기존 전세 분석 결과 내 섹션 추가 | 사용자 동선 자연스러움 |
| 보증료율 관리 | 별도 config 객체 | 기관 기준 변경 시 config만 수정 |

### 8.3 파일 구조

```
lib/
  guarantee-insurance.ts        # 규칙 엔진 + 보증료 계산
app/(app)/jeonse/analysis/
  page.tsx                      # 기존 전세 분석에 보증보험 섹션 추가
components/results/
  GuaranteeInsuranceCard.tsx    # 보증보험 판단 결과 카드 컴포넌트
```
