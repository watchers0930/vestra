# VESTRA Feasibility Report (SCR 수준 사업성 분석) 고도화 기획서

## 1. 기능 개요
회사가 제출한 다수 자료(투자계획서, 재원조달계획, 사업계획서, 재무분석표 등)를 근거로 사업성을 정밀 검토하고, 공공데이터와 교차 검증하여 서울신용평가(SCR) 수준의 전문적인 분석 보고서를 생성하는 기능입니다. 단순 요약을 넘어 제출 데이터의 '합리성'을 판정하고 검증 의견을 제시하는 것이 핵심입니다.

## 2. 사용자 스토리
- **시행사/CM 담당자**: "우리가 세운 사업 수지가 시장 실거래가나 표준 공사비 대비 얼마나 현실적인지 객관적으로 검증받고 싶다."
- **PF 대출 심사역**: "시행사가 제출한 장밋빛 전망(높은 분양가, 낮은 공사비)을 공공데이터 기반으로 필터링하고 리스크를 파악하고 싶다."
- **투자 결정권자**: "사업의 각 단계별로 전문 평가사의 의견과 같은 날카로운 '검증 의견'이 포함된 보고서가 필요하다."

## 3. 핵심 강화 기능 (5대 요구사항 반영)

### (1) 다중 문서 입력 및 통합 파싱
- **멀티 파일 업로드**: 투자계획서(PDF), 재원조달계획(DOCX), 사업계획서(HWP), 재무분석표(XLSX/PDF) 등 복수 파일 동시 수용.
- **컨텍스트 병합**: 여러 파일에 흩어진 수치(예: A파일의 총공사비, B파일의 분양 예정가)를 하나의 사업성 모델로 통합 매핑.
- **데이터 상충 감지**: 파일 간 수치가 다를 경우(예: 사업계획서와 재무분석표의 연면적 차이) 경고 및 사용자 확인 유도.

### (2) 주장-검증 엔진 (Claim-Verification Engine)
- **주장 추출**: 업체 제출 자료에서 핵심 수치 자동 추출 (분양가, 공사비, 분양률, 기대 수익률 등).
- **교차 검증 소스**:
    - **분양가**: `molit-api.ts` 기반 인근 1km 내 최신 실거래가 및 인근 분양가.
    - **공사비**: 한국건설기술연구원(KICT) 건설공사비지수 및 용도별 표준 단가.
    - **분양률**: `supply-api.ts` 및 `reb-api.ts` 기반 해당 지역 미분양 현황 및 인근 단지 초기 분양률.
    - **인구/수요**: 행안부 인구통계 및 `seoul-data-api.ts` 기반 유동인구/상권 데이터.

### (3) 합리성 판단 로직 (Rationality Judgment)
- **판단 지표**: 제출 수치가 벤치마크 대비 어느 범위에 있는지 계산.
    - **적정 (Appropriate)**: 시장 평균 ±10% 이내.
    - **낙관적 (Optimistic)**: 시장 평균 대비 10~25% 유리한 수치 (근거 필요).
    - **비현실적 (Unrealistic)**: 시장 평균 대비 25% 이상 괴리 (위험 신호).
- **의견 생성**: 각 항목별로 "인근 실거래가 대비 분양가가 과도하게 높게 책정됨" 등의 논리적 근거 제시.

### (4) 업계 벤치마크 DB (Benchmark DB)
- **표준 기준점**: 내부 및 외부 데이터를 활용한 기준 DB 구축.
    - **건설공사비지수**: 월별 변동 추이 반영.
    - **평균 분양률**: 지역별/용도별(아파트/오피스텔/상가) 3개년 평균.
    - **PF 금리 범위**: 신용등급별/담보물별 시장 금리 밴드.
    - **Cap Rate**: 수익형 부동산의 지역별 목표 수익률 기준.

### (5) 검증 의견서 (SCR Style Verification)
- **섹션별 검토 의견**: 보고서 각 장(Chapter) 하단에 '검토 의견(Review Opinion)' 섹션 강제 추가.
- **SCR 스타일 레이아웃**: 요약-본문-검증의견-종합판정 순의 전문 보고서 형식.
- **리스크 하이라이트**: 비현실적인 수치가 발견된 섹션은 붉은색 강조 및 대응 방안 제언.

## 4. 상세 기능 구성 (보안된 보고서 구조)

| 장(Chapter) | 주요 내용 | 검증 포인트 (Verification) |
| :--- | :--- | :--- |
| **I. 사업 개요** | 위치, 규모, 시설계획 | 건축물대장상 대지 정보와 일치 여부 |
| **II. 시장 환경** | 지역 수급, 시세 추이 | 입주물량 과잉 여부, 인구 감소 추세 확인 |
| **III. 분양가 분석** | 계획 분양가, 가격 전략 | **[핵심]** 인근 실거래가 대비 적정성 판정 |
| **IV. 공사비/사업비** | 직접비, 간접비, 금융비용 | **[핵심]** 건설공사비지수 대비 과소계정 여부 |
| **V. 재무 타당성** | 현금흐름, 수익률, BEP | 분양 지연 시나리오에 따른 상환 가능성 |
| **VI. 종합 의견** | 투자 적격 등급 (V-Score) | 각 섹션 의견을 종합한 합리성 판정 |

## 5. 기술적 요구사항

### 데이터 및 알고리즘
- **`lib/feasibility-validator.ts`**: (신규) 업체 주장 수치와 공공데이터를 비교하여 3단계 합리성 등급을 매기는 엔진.
- **`lib/benchmark-db.ts`**: (신규) 건설공사비지수, 금리 밴드 등 기준 데이터를 관리하는 모듈.
- **OpenAI gpt-4o-mini**: 다중 문서 컨텍스트 병합 및 SCR 스타일의 날카로운 비평 의견 생성.
- **Prompt Engineering**: "심사역의 관점에서 수치의 허점을 지적하라"는 역할 부여(Persona).

### 백엔드/프런트엔드
- **파일 처리**: `unpdf` 및 `mammoth`를 활용한 다중 문서 파싱 스트림 처리.
- **Next.js 16 (React 19)**: 멀티 파일 업로드 Progress UI 및 실시간 검증 결과 프리뷰.
- **Prisma/Neon**: 다중 파일 연관 관계 및 검증 이력 저장.

## 6. 우선순위 및 로드맵

| 단계 | 목표 | 주요 내용 |
| :--- | :--- | :--- |
| **Phase 1** | 통합 파싱 엔진 | 다중 파일 업로드 UI + 문서 간 데이터 컨텍스트 병합 및 상충 감지 |
| **Phase 2** | 주장-검증 시스템 | MOLIT/KICT 데이터 연동 + 주요 지표(분양가/공사비) 교차 검증 |
| **Phase 3** | 합리성 판단 로직 | 벤치마크 DB 구축 + 3단계(적정/낙관/비현실) 판단 엔진 구현 |
| **Phase 4** | 전문 보고서 완성 | SCR 스타일 PDF 렌더링 + 각 장별 검증 의견 섹션 자동 생성 |

## 7. 기존 코드베이스 활용 계획
- **`lib/validation-engine.ts`**: 데이터 신뢰도 평가 프레임워크 재사용.
- **`lib/v-score.ts`**: 다중 소스 가중치 합산 알고리즘을 사업성 등급 산출에 응용.
- **`lib/molit-api.ts`**: 실거래가 기반 분양가 검증의 핵심 소스로 활용.
- **`components/forms`**: 멀티 파일 드롭존 및 슬라이더 컴포넌트 활용.
