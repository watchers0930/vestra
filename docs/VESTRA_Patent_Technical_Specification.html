1. 기술 개요 (Executive Summary)

VESTRA는 한국 부동산 시장에 특화된 AI 자산관리 플랫폼으로, 등기부등본 자동 파싱 → 리스크 스코어링 → 시세 추정 → 가치 전망 → 계약서 분석 → 데이터 검증의 6단계 통합 분석 파이프라인을 구현합니다.

핵심 기술적 독창성

  1. 2단계 하이브리드 AI 아키텍처: Phase 1 (규칙 엔진, LLM 비의존) → Phase 2 (LLM 종합 의견). LLM 호출량 ~70% 절감.
  2. 12-Factor 가중치 감점 리스크 모델: 100점 만점에서 12가지 위험 요인별 차등 감점. 4단계 심각도(critical/high/medium/low) 체계.
  3. 3-Tier 비교매물 분석: 건물명 매칭 → 면적 유사도 → 지역 평균의 계층적 Fallback. 시간 감쇠 가중치 e^(-0.1t) 적용.
  4. 4-Layer 검증 엔진: 포맷/산술/문맥/크로스체크의 4단계 독립 검증. 50개 이상 검증 규칙.
  5. 다중 시나리오 세금 엔진: 3가지 정책 시나리오(유예/연장/복원)별 세금 시뮬레이션.

3. 핵심 알고리즘 #1: 등기부등본 자동 파싱 엔진

3.1 기술적 과제

한국 부동산 등기부등본은 법원등기소마다 서식이 다르고, 인터넷등기소/OCR 추출 텍스트 등 다양한 입력 형태가 존재합니다. VESTRA 파싱 엔진은 3가지 입력 형식(인터넷등기소, 법원등기, OCR)을 단일 파이프라인으로 처리합니다.

3.3 핵심 알고리즘 상세

3.3.1 다형 날짜 추출 (extractDate)

한국 등기부등본에서 사용되는 다양한 날짜 형식을 통일된 YYYY.MM.DD 포맷으로 변환합니다.

// 지원 포맷: "2023년 5월 15일", "2023년05월15일", "2023.05.15", "2023-05-15" function extractDate(text: string): string { const m1 = text.match(/(\d{4})\s*년\s*(\d{1,2})\s*월\s*(\d{1,2})\s*일/); if (m1) return `${m1[1]}.${m1[2].padStart(2,"0")}.${m1[3].padStart(2,"0")}`; const m2 = text.match(/(\d{4})[.\-/](\d{1,2})[.\-/](\d{1,2})/); if (m2) return `${m2[1]}.${m2[2].padStart(2,"0")}.${m2[3].padStart(2,"0")}`; return ""; }

4. 핵심 알고리즘 #2: 12-Factor 가중치 리스크 스코어링

4.1 스코어링 모델 개요

Score = max(0, 100 - Σi=112 Di)     where Di = deduction(factori, severityi)

100점 만점 감점 방식으로, 높은 점수 = 안전, 낮은 점수 = 위험을 의미합니다. 12개 위험 요인을 독립적으로 평가하여 합산합니다.

4.4 근저당 비율 평가 알고리즘

mortgageRatio = (totalMortgageAmount / estimatedPrice) × 100
// 5단계 임계값 기반 비선형 감점 if (ratio > 120%) → 30pt (critical: "깡통주택 매우 위험") if (ratio > 100%) → 25pt (critical: "시세 초과") if (ratio > 80%) → 20pt (high: "보증금 회수 위험") if (ratio > 70%) → 10pt (medium: "주의 필요") if (ratio > 50%) → 5pt (low: "모니터링") else → 0pt (안전)

4.5 요약 자동 생성

최종 점수, 등급, 위험 요소 수, 갑/을구 현행 건수를 종합하여 자연어 요약을 자동 생성합니다.

5. 핵심 알고리즘 #3: 3-Tier 비교매물 시세 추정

5.1 3-Tier Fallback 전략

Tier 1: 건물명 키워드 매칭 ├─ 매칭 성공 (N > 0) → ★ 이 Tier 사용, 면적 추가 필터(±30%) └─ 매칭 실패 (N = 0) ─▶ Fallback │ Tier 2: 면적 유사도 매칭 (±20%) ├─ 매칭 성공 (N > 0) → ★ 이 Tier 사용 └─ 매칭 실패 (N = 0) ─▶ Fallback │ Tier 3: 구/군 전체 평균 └─ 모든 거래 데이터 사용 → ★ 최후 수단

5.2 가중치 체계

5.2.1 시간 감쇠 가중치 (Exponential Decay)

wtime(t) = e-0.1 × t     (t: 경과 개월 수)

최근 거래에 높은 가중치를 부여합니다. 3개월 전 거래: 0.74, 6개월: 0.55, 12개월: 0.30.

5.2.2 면적 유사도 가중치

warea = 1 - min(|txArea - targetArea| / targetArea, 0.5)

5.2.3 면적 보정 가격

adjustedPrice = dealAmount × (targetArea / txArea)     (면적 차 ≥ 5%인 경우)

5.2.4 층수 프리미엄

adjustedPrice = price × (1 + 0.005 × (targetFloor - txFloor))

층당 0.5% 프리미엄/디스카운트를 적용합니다.

5.3 신뢰도 산출

구성 요소최대 점수산출 기준
비교매물 수50점건당 10점 (최대 5건 = 50점)
데이터 최근성20점3개월 이내=20pt, 6개월=10pt, 12개월=5pt
가격 분산도20점CV < 0.15 → 20pt, CV < 0.30 → 10pt
건물명 매칭10점Tier 1 사용 시 보너스
합계최대 95점Cap: 95 (추정은 본질적 불확실성 포함)

5.4 가중 평균 최종 가격

estimatedPrice = Σ(adjustedPricei × wi) / Σ(wi)     where wi = wtime × warea

6. 핵심 알고리즘 #4: 선형회귀 + 3-시나리오 자산가치 전망

6.1 추세 분석 (Linear Regression)

y = slope × x + intercept     (x: 경과 개월, y: 거래가)
R² = 1 - SSres / SStot
annualGrowthRate = (slope × 12) / avgPrice     (cap: -20% ~ +30%)

최소자승법(OLS)으로 거래가격의 시간적 추세를 산출하고, R² 값으로 추세의 일관성을 평가합니다.

6.2 3-시나리오 모델링

기본 시나리오 (Base)

baseRate = trend × 0.7 + longTermAvg × 0.3     (데이터 3건 이상)

데이터 부족 시 장기 평균 성장률(3%) 사용. 장기 예측에는 평균회귀(mean reversion) 감쇠 적용:

  • 1년: baseRate (감쇠 없음)
  • 5년: baseRate × 0.85 (15% 평균회귀)
  • 10년: baseRate × 0.7 + inflation × 0.3 (30% 평균회귀 + 인플레이션)

낙관 시나리오 (Optimistic)

optimisticRate = baseRate × 1.3     (하한: inflation + 2%p)

비관 시나리오 (Pessimistic)

pessimisticRate = baseRate × 0.5     (하한: 1y=-5%, 5y=-3%, 10y=-1%)

복리 성장 적용

P(t) = P0 × (1 + r)t

8. 핵심 알고리즘 #6: 4-Layer 데이터 검증 엔진

8.1 4-Layer 검증 아키텍처

Layer A1: 포맷 & 타입 검증 (Format Validation) ├─ 날짜 형식 (YYYY.MM.DD, 범위 1900~2035) ├─ 금액 형식 (양수, 100만원~500억원) ├─ 순위번호 순차성 (중복/음수 검사) ├─ 권리자명 (2~30자) ├─ 위험유형 열거값 (danger|warning|safe|info) └─ 섹션 완전성 (주소, 갑구 존재) │ ▼ Layer A2: 합계 & 산술 검증 (Arithmetic Validation) ├─ 근저당 합계 = Σ(활성 근저당) ├─ 전세권 합계 = Σ(활성 전세권) ├─ 총채권액 = 근저당 + 전세 ├─ 활성 건수 = 전체 - 말소 ├─ 소유권이전 횟수 └─ 근저당비율 = 근저당/시세 × 100 │ ▼ Layer A3: 문맥 & 규칙 검증 (Context Validation) ├─ 등기 날짜 시간순 정합성 ├─ 말소등기 ↔ 원본 대응 관계 ├─ 소유권 체인 일관성 ├─ 압류 후 근저당 설정 탐지 ├─ 신탁-근저당 충돌 탐지 ├─ 갑구 1번 = 소유권보존 규칙 └─ 소유권 없이 을구만 존재 경고 │ ▼ Layer A4: 크로스체크 검증 (Cross-Check Validation) ├─ 요약 불리언 플래그 vs 실제 엔트리 ├─ 추정가격 합리성 (5천만~500억) ├─ 리스크 스코어 팩터 일치성 ├─ 총감점 = Σ(팩터별 감점) ├─ 점수 = max(0, 100 - 총감점) └─ AI 의견 관련성 (소프트 체크)

12.2 VESTRA의 6가지 기술적 독창성

독창성 #1: 2단계 하이브리드 AI 아키텍처

기존 PropTech AI는 전체 분석을 LLM에 의존하여 비용이 높고 재현성이 낮습니다. VESTRA는 규칙 엔진(Phase 1)으로 정량 분석을 완료한 후, LLM(Phase 2)은 자연어 종합 의견만 생성하여 비용 70% 절감과 100% 재현성을 동시에 달성합니다.

독창성 #2: 12-Factor 가중치 감점 리스크 모델

부동산 등기부등본의 12가지 위험 요인을 4단계 심각도로 분류하고, 비선형 임계값 기반 감점 체계를 적용합니다. 특히 근저당비율의 5단계 임계값(50/70/80/100/120%)과 건수 비례 감점(압류 건당 25pt)은 도메인 전문가 지식을 정량화한 고유한 모델입니다.

독창성 #3: 3-Tier 비교매물 Fallback + 지수 감쇠 가중치

건물명 매칭(정확도 높음) → 면적 유사도(범용성) → 지역 평균(최후 수단)의 계층적 Fallback은 데이터 가용성에 따라 최적의 추정을 보장합니다. 시간 감쇠 가중치 e^(-0.1t)는 최근 거래를 우선하면서도 과거 데이터를 완전히 배제하지 않는 균형을 제공합니다.

독창성 #4: 평균회귀 + 인플레이션 블렌딩 장기 전망

단기(1년)는 추세를, 장기(10년)는 평균회귀와 인플레이션을 블렌딩하는 적응적 예측 모델입니다. 추세의 70%와 장기 평균의 30%를 혼합하고, 5년/10년에 각각 15%/30% 평균회귀 감쇠를 적용합니다.

독창성 #5: 4-Layer 독립 검증 엔진

포맷→산술→문맥→크로스체크의 4단계 독립 검증은 부동산 분석에서 유일한 접근법입니다. 특히 갑구-을구 간 교차 검증(압류 후 근저당, 신탁-근저당 충돌)은 단일 섹션 분석으로는 발견할 수 없는 복합 위험을 탐지합니다.

독창성 #6: 다중 정책 시나리오 세금 시뮬레이션

TaxConfig 인터페이스를 통해 정책 변수(다주택 중과, 생애최초 감면, 공정시장가액비율)를 매개변수화하여, 정책 변경 시나리오별 세금 영향을 즉시 시뮬레이션할 수 있습니다. 이는 투자 의사결정에 정책 불확실성을 반영하는 유일한 접근법입니다.
