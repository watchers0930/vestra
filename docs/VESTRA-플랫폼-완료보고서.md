## 1. 프로젝트 개요

### 1.1 플랫폼 소개
VESTRA는 **AI 기반 부동산 자산관리 플랫폼**으로, 등기부등본 권리분석, 전세사기 예방, 시세전망, 계약서 AI 검토, 세무 시뮬레이션 등 부동산 거래의 전 과정에서 AI 분석을 제공한다.

### 1.2 개발 목표
1. 전세사기 등 부동산 거래 피해를 사전에 예방하는 AI 위험 진단 시스템 구축
2. 등기부등본, 실거래가, 공공데이터를 통합 분석하는 원스톱 부동산 분석 플랫폼 제공
3. 비전문가도 부동산 권리관계를 이해할 수 있는 직관적 리포트 생성

### 1.3 서비스 URL
- **프로덕션**: https://vestra-plum.vercel.app
- **GitHub**: bmicns/vestra (Private)

### 6.2 파라미터 검증 현황

> **중요**: 자체 설계 파라미터(가중치, 감점 수치, 증폭계수 등)는 **도메인 전문가 휴리스틱에 기반한 초기값**이며, 실증 데이터를 통한 캘리브레이션이 아직 완료되지 않은 상태이다.

| 항목 | 근거 | 검증 상태 |
|-----|------|----------|
| 세금 세율/공제액 | 한국 세법 (법적 근거 있음) | 검증 완료 |
| 통계 모델 (ARIMA, Holt 등) | 학술 표준 방법론 | 방법론 검증 완료, 파라미터 튜닝 필요 |
| 감점 수치 (30/25/20점 등) | 전문가 휴리스틱 | **미검증 - 실증 데이터 필요** |
| 증폭계수 (1.3~1.7) | 전문가 휴리스틱 | **미검증 - 실증 데이터 필요** |
| V-Score 가중치 | 전문가 휴리스틱 | **미검증 - 실증 데이터 필요** |
| 15-피처 가중치 | 전문가 휴리스틱 | **미검증 - 실증 데이터 필요** |
| 시계열 패턴 조건 | 전문가 휴리스틱 | **미검증 - 실증 데이터 필요** |

### 6.3 검증 로드맵

- **Phase 1** (1~2개월): HUG 보증사고, 법원 경매, 전세사기 판례 데이터 수집
- **Phase 2** (2~3개월): 후향적 검증 (목표: 민감도 80%+, 오경보율 20% 이하)
- **Phase 3** (1~2개월): Thompson Sampling 기반 자동 캘리브레이션 + 전문가 자문
- **Phase 4** (지속): 실서비스 A/B 테스트 및 분기별 리뷰

1. **파라미터 실증 미검증**: 위험도 스코어링의 가중치·감점 수치가 실제 사고 데이터로 검증되지 않음

## 10. 결론

VESTRA는 **8개의 자체 개발 AI/ML 엔진**과 **6개의 외부 공공 API**를 통합하여, 부동산 거래의 전 과정에서 AI 기반 위험 분석을 제공하는 플랫폼으로서 핵심 기능 구현이 완료된 상태이다.

다만, **자체 설계 파라미터의 실증 검증이 가장 긴급한 과제**이며, 이는 서비스 신뢰성과 특허 출원의 전제 조건이다. 세법 기반 수식과 학술 표준 방법론은 검증이 완료된 반면, 위험도 스코어링·사기 진단·시세 예측의 구체적 가중치는 실제 데이터를 통한 캘리브레이션이 필요하다.
