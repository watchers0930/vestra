## 1. 프로젝트 개요

VESTRA는 한국 부동산 시장에 특화된 AI 기반 부동산 분석 플랫폼이다. 전세 안전성 분석, 등기부등본 분석, 계약서 검토, 세금 계산, 시세 예측 등 부동산 거래의 핵심 리스크를 종합적으로 분석한다.

### 1.1 핵심 가치

- **자체 알고리즘 기반 분석**: 핵심 분석 엔진(등기 파서, 검증 엔진, 리스크 스코어링)은 LLM에 의존하지 않는 순수 TypeScript 구현 (1,803 LOC)
- **V-Score**: 5개 정보원을 가중 융합한 통합 위험 지수 (특허 출원 대상)
- **사기 위험 모델**: 15개 특성을 활용한 Gradient Boosting 시뮬레이션 (규칙 기반 앙상블)
- **운영 준비 완료**: RBAC, 감사 로그, 비용 관리, 보안 헤더 등 상용 수준 인프라

## 4. 핵심 알고리즘

### 4.1 등기부 파서 (registry-parser.ts, 598 LOC)

LLM 없이 순수 정규표현식과 파싱 로직으로 비정형 한글 등기부 텍스트를 구조화한다.

**처리 파이프라인**:
1. **텍스트 정규화**: CRLF→LF 변환, 탭 처리
2. **섹션 분리**: 표제부/갑구/을구 감지 (3개 패턴 폴백)
3. **멀티라인 누적**: 여러 줄에 걸친 항목을 하나로 통합
4. **필드 추출**: 18종 한국어 권리 유형 인식 (소유권이전, 근저당, 가압류 등)
5. **한글 금액 변환**: 6가지 표기 패턴 지원 ("금 4억8천만원", "금480,000,000원" 등)
6. **말소 감지**: 말소된 권리 자동 필터링

**출력**: `ParsedRegistry` (표제부, 갑구[], 을구[], 요약)

### 4.2 검증 엔진 (validation-engine.ts, 1,232 LOC)

파싱된 데이터의 품질과 정합성을 4단계로 검증한다.

| 단계 | 검증 내용 | 예시 |
|------|----------|------|
| 1. 포맷 검증 | 날짜/금액 형식 | "2024-01-15" 형식 확인 |
| 2. 논리 검증 | 소유권 일관성 | 매도인-매수인 체인 검증 |
| 3. 위험 패턴 | 레드플래그 감지 | 단기 소유권 이전, 금액 급변 |
| 4. 신뢰도 전파 | 확신 점수 산출 | 파싱 결과별 신뢰도 계산 |

### 4.3 리스크 스코어링 (risk-scoring.ts, 937 LOC)

100점 스케일(100=안전, 0=위험)의 종합 위험 점수를 산출한다.

**12개 위험 요소**:

| 요소 | 가중치 | 위험 기준 |
|------|-------|----------|
| 근저당 비율 | 높음 | 감정가 대비 120% 초과 시 위험 |
| 압류/가압류 | 높음 | 존재 시 즉시 위험 |
| 우선변제권 비율 | 높음 | 보증금 대비 높을수록 위험 |
| 소유권 이전 빈도 | 중간 | 단기 빈번 이전 시 위험 |
| 전세가율 | 중간 | 시세 대비 80% 초과 시 주의 |
| 건물 노후도 | 중간 | 30년 초과 시 감점 |
| 다주택 지표 | 중간 | 임대인 다주택 보유 시 |
| 지역 사기율 | 중간 | 해당 지역 사기 발생 빈도 |
| 체납 이력 | 중간 | 세금 체납 존재 시 |
| 건물 상태 | 낮음 | 위반건축물 등 |
| 공실률 | 낮음 | 지역 공실률 높을 시 |
| 가격 변동성 | 낮음 | 급등/급락 이력 |

**등급 체계**: A (85+) | B (70-84) | C (50-69) | D (30-49) | F (30 미만)

### 4.4 V-Score (v-score.ts, 850 LOC) — 특허 핵심

이질적 5개 정보원을 가중 융합하여 단일 통합 위험 지수(0-100)를 산출한다.

**정보원 및 가중치**:

| 정보원 | 가중치 | 분석 대상 |
|--------|-------|----------|
| 등기 권리 | 30% | 소유권, 근저당, 가압류 |
| 가격/전세 비율 | 25% | 시장 안정성 |
| 계약 조건 | 20% | 계약 리스크 |
| 임대인 지표 | 15% | 임대인 재무 위험 |
| 지역 위험 | 10% | 사기 발생 빈도 |

**비선형 상호작용 규칙** (7개):
- 복합 증폭: 등기_저 + 가격_저 → -10점
- 교차 완화: 계약_안전 + 가격_안정 → +3점
- 연쇄 감지: 전체_고위험 → -10점 추가 감점

### 4.5 사기 위험 모델 (fraud-risk-model.ts, 680 LOC) — 특허 특징

15개 특성을 활용한 규칙 기반 Gradient Boosting 시뮬레이션이다.

**특성 그룹**:
- **권리 (4)**: 근저당 비율, 압류 수, 우선변제, 가압류
- **가격 (3)**: 전세가율, 변동성, 공실률
- **임대인 (4)**: 다주택, 법인, 체납, 신용점수
- **건물 (2)**: 노후도, 세대수
- **지역 (2)**: 사기율, 경매율

**SHAP 유사 기여도**: Leave-One-Out 방식으로 각 특성의 기여도를 산출하여 설명 가능한 결과 제공

## 5. 데이터베이스 설계

### 5.1 모델 구성 (25개 Prisma 모델)

**인증 영역**:
- `User`, `Account`, `Session`, `VerificationToken` — NextAuth v5 표준 스키마

**핵심 데이터**:
- `Analysis` — 분석 결과 저장 (유형, 주소, 사기 위험도, V-Score)
- `Asset` — 사용자 자산 포트폴리오 (시세, 안전/위험 점수)

**요금제 및 결제**:
- `Subscription` — 플랜 관리 (FREE | PRO | BUSINESS)
- `Payment` — 결제 기록

**모니터링**:
- `MonitoredProperty` — 등기 변동 감시 목록
- `MonitoringAlert` — 변동 알림
- `VerificationRequest`, `SharedReport` — P2P 교차 검증

**관리자 및 감사**:
- `AuditLog` — 완전한 감사 추적 (사용자, 행위, IP, 타임스탬프)
- `SystemSetting` — AES-256-GCM 암호화 설정 저장
- `Announcement` — 시스템 공지

**ML 학습 데이터**:
- `TrainingData` — 암호화된 등기 텍스트 + 파싱 결과 JSON (NLP 모델 학습용)
- `DomainVocabulary` — 한국 부동산 용어 사전
- `FraudCase` — 사기 사례 DB (위치 기반 히트맵용)

### 5.2 사용량 제어

- `RateLimit` — 분당 슬라이딩 윈도우 (DB 기반)
- `DailyUsage` — 역할별 일일 사용량 카운터

## 8. 주요 기능

### 8.1 전세 안전 분석

전세 보증금의 안전성을 종합 평가한다. 전세가율 계산, 보증금 회수 시뮬레이션, 임대인 위험 평가, 지역 사기 히트맵을 제공한다. 전입신고/확정일자/전세권 설정 체크리스트로 실무 가이드도 포함한다.

### 8.2 등기부등본 분석

비정형 등기부 텍스트를 자동 파싱하여 25개 이상의 위험 요소를 식별한다. 소유권 체인 분석, 근저당/가압류 충돌 감지, 신뢰도 점수를 산출한다.

### 8.3 계약서 분석

매매/임대 계약서의 조항별 위험도를 AI로 평가한다. 누락 조항을 감지하고 관련 판례를 인용하여 법적 근거를 제시한다.

### 8.4 세금 계산기

취득세(다주택 중과 포함), 양도소득세(보유 기간 혜택), 종합부동산세(공시가격 기준)를 2024-2026 시나리오별로 계산한다.

### 8.5 시세 예측

3시나리오 예측(상승/기본/하락), 유사 매물 매칭, 지역 트렌드 분석을 제공한다. Kakao Maps/Leaflet 기반 히트맵으로 시각화한다.

### 8.6 AI 부동산 어시스턴트

법률 Q&A, 판례 검색 연동, 계약 검토 가이드를 제공하는 AI 챗봇이다. 분당 30회 요청 제한으로 비용을 관리한다.

### 8.7 관리자 대시보드

사용자 관리(RBAC, 인증), 시스템 통계, 감사 로그, ML 학습 데이터 관리(업로드/검토/승인/내보내기), 도메인 용어 사전, 공지 관리 기능을 포함한다.

## 12. 핵심 기술적 의사결정

| 결정 | 근거 |
|------|------|
| 핵심 분석에 LLM 미사용 | 비용 예측 가능성, 응답 일관성, 오프라인 가용성 확보 |
| DB 기반 요청 제한 | 서버리스 인스턴스 간 정확한 카운트 보장 |
| JWT 암호화 (JWE) | Edge 미들웨어에서 Prisma 없이 인증 처리 |
| AES-256-GCM | 민감 학습 데이터 및 시스템 설정 보호 |
| 규칙 기반 Gradient Boosting | ML 라이브러리 의존성 없이 설명 가능한 사기 탐지 |
| V-Score 가중 융합 | 이질적 데이터를 단일 지표로 통합, 비선형 상호작용 반영 |

## 14. 결론

VESTRA는 18일 간의 집중 개발을 통해 31,315 LOC 규모의 상용 수준 부동산 분석 플랫폼을 구축했다. 핵심 분석 엔진(등기 파서, 검증 엔진, 리스크 스코어링, V-Score, 사기 위험 모델)은 LLM에 의존하지 않는 자체 알고리즘으로 구현하여 비용 효율성과 결과 일관성을 확보했다. TypeScript strict mode, 종합 테스트, 보안 우선 설계 등 높은 엔지니어링 품질을 유지하며, RBAC/감사 로그/비용 관리 등 상용 운영에 필요한 인프라를 갖추고 있다.
