## 2. 특허기술 A: 등기부등본 기반 다차원 위험도 스코어링 시스템

### 2.1 기술 명칭
**등기부등본 권리관계의 다차원 감점 및 비선형 상호작용 증폭 기반 부동산 위험도 자동 평가 시스템**

### 2.2 기술 설명
등기부등본의 갑구(소유권)·을구(제한물권) 정보를 정량적으로 분석하여 100점 만점 기준 감점 방식으로 위험도를 산출하며, **13가지 개별 위험 팩터**와 **6가지 비선형 상호작용 증폭 규칙**을 결합한다.

### 2.3 수식

#### 2.3.1 기본 위험도 점수

```
S_total = max(0, 100 - D_total)

D_total = Σ(i=1→13) D_i + D_interaction + D_temporal
```

여기서:
- `S_total`: 최종 안전 점수 (0~100)
- `D_total`: 총감점
- `D_i`: 개별 위험 팩터 감점
- `D_interaction`: 상호작용 증폭 감점
- `D_temporal`: 시계열 이상 감점

#### 2.3.2 근저당비율 감점 함수

```
R_mortgage = (M_total / P_est) × 100

       ┌ 30,  R > 120%
       │ 25,  100% < R ≤ 120%
D_1 =  │ 20,  80% < R ≤ 100%
       │ 10,  70% < R ≤ 80%
       │ 5,   50% < R ≤ 70%
       └ 0,   R ≤ 50%
```

여기서:
- `M_total`: 총근저당액 = Σ(을구 내 활성 근저당권 채권최고액)
- `P_est`: 추정시세

#### 2.3.3 총채권비율 감점 함수

```
R_claims = ((M_total + J_total) / P_est) × 100

         ┌ 25,  R > 100%  (보증금 전액 미회수 위험)
D_8 =    │ 15,  80% < R ≤ 100%
         │ 8,   60% < R ≤ 80%
         └ 0,   R ≤ 60%
```

여기서:
- `J_total`: 총전세권액 = Σ(을구 내 활성 전세권 설정액)

#### 2.3.4 비선형 상호작용 증폭

6가지 상호작용 규칙에 대해:

```
D_interaction = Σ(k=1→6) max(0, (D_a(k) + D_b(k)) × (α_k - 1))
```

| 규칙 k | 팩터 a | 팩터 b | 증폭계수 α_k | 의미 |
|--------|--------|--------|-------------|------|
| 1 | 압류 | 경매개시결정 | 1.5 | 강제처분 절차 진행 |
| 2 | 다중근저당 | 고근저당비율 | 1.3 | 과잉 레버리지 |
| 3 | 신탁 | 다중근저당 | 1.4 | 신탁 내 채무 과다 |
| 4 | 압류 | 임차권등기 | 1.6 | 임차인 최악 시나리오 |
| 5 | 경매 | 초과채권(>120%) | 1.7 | 회수 불가능 상태 |
| 6 | 압류 | 가처분 | 1.3 | 복수 법적 분쟁 |

#### 2.3.5 등급 판정

```
Grade(S) = { A: S ≥ 85, B: 70 ≤ S < 85, C: 50 ≤ S < 70, D: 30 ≤ S < 50, F: S < 30 }
```

### 2.4 선행기술 조사

| 선행기술 | 출처 | 차이점 |
|---------|------|--------|
| 지지옥션 경매 낙찰가 예측 (등록번호 10-2130104) | 유전자 알고리즘 기반 경매 낙찰가 예측 | 등기부 권리관계 위험도 스코어링이 아닌 경매 낙찰가 예측에 한정 |
| CoreLogic US20140025548A1 | 부동산 거래 이상 탐지 자동화 | 거래 패턴 이상 탐지에 초점, 등기부 갑구/을구 기반 다차원 감점 시스템 없음 |
| AI 권리분석 법률서비스 (법률신문 보도) | 등기부등본 기반 AI 분석 | 상세 수식 및 비선형 상호작용 증폭 메커니즘 공개 없음 |

**차별성**: 본 기술은 13가지 개별 팩터 + 6가지 비선형 상호작용 증폭 규칙을 결합한 **복합 위험도 산출 방법론**이 핵심이며, 특히 팩터 간 상호작용이 단순 합산이 아닌 **증폭계수(α)를 적용한 비선형 결합**이라는 점에서 기존 선행기술과 구별됨.

### 2.5 파라미터 근거 및 검증 계획

> **현재 상태**: 감점 수치(30/25/20/10/5점), 증폭계수(1.3~1.7), 등급 기준점(85/70/50/30점) 등은 **부동산 도메인 전문가의 휴리스틱 판단에 기반한 초기값**이며, 실증 데이터를 통한 캘리브레이션이 완료되지 않은 상태이다.

**검증 계획**:
1. **실증 데이터 수집**: HUG(주택도시보증공사) 전세보증사고 데이터, 법원 경매 낙찰 데이터, 전세사기 피해 판례 DB를 수집하여 "위험 물건"과 "안전 물건"의 라벨링 데이터셋 구축
2. **후향적 검증(Backtesting)**: 과거 사고 물건의 등기부등본을 모델에 입력하여, 사고 발생 전 시점에서 모델이 해당 물건을 고위험(D/F 등급)으로 분류하는 비율(민감도) 측정
3. **파라미터 최적화**: 적응형 가중치 튜닝 시스템(특허기술 F)을 활용하여 Brier Score 최소화 기준으로 감점 수치 및 증폭계수를 데이터 기반 자동 튜닝
4. **전문가 자문**: 법무사·감정평가사·부동산 변호사 패널의 정성적 검토를 통해 감점 구간 및 등급 기준의 타당성 검증
5. **A/B 테스트**: 실제 서비스에서 기존 파라미터와 캘리브레이션된 파라미터의 사용자 만족도 및 예측 정확도 비교

## 3. 특허기술 B: 5-모델 앙상블 부동산 시세 예측 엔진

### 3.1 기술 명칭
**선형회귀·평균회귀·모멘텀·ARIMA·ETS 5-모델 앙상블 기반 부동산 시세 예측 및 신뢰도 전파 시스템**

### 3.2 기술 설명
단일 모델의 한계를 극복하기 위해 5가지 이질적 예측 모델을 앙상블하고, 데이터 품질에 기반한 **신뢰도 점수**를 함께 산출한다.

### 3.3 수식

#### 3.3.1 모델 1: 선형회귀 추세 (Least Squares)

```
ŷ = β₁x + β₀

β₁ = (nΣxᵢyᵢ - ΣxᵢΣyᵢ) / (nΣxᵢ² - (Σxᵢ)²)
β₀ = (Σyᵢ - β₁Σxᵢ) / n

R² = 1 - (Σ(yᵢ - ŷᵢ)² / Σ(yᵢ - ȳ)²)

연간성장률 g = clamp((β₁ × 12) / ȳ, -0.20, 0.30)
```

#### 3.3.2 모델 2: 평균회귀 (Mean Reversion)

```
deviation = (P_current - P_mean) / P_mean
reversion_rate = -0.3 × deviation

P̂_t = P_current × (1 + reversion_rate)^t
```

여기서:
- `P_mean`: 전체 기간 평균 거래가
- `-0.3`: 평균회귀 계수 (연간 30% 회귀)

#### 3.3.3 모델 3: 모멘텀 (지수 감쇠)

```
g_recent = 최근 6개월 거래가 성장률

P̂_t = P_current × (1 + g_recent × e^(-0.15t))^t
```

여기서:
- `e^(-0.15t)`: 지수 감쇠 함수로 장기 예측 시 모멘텀 효과 축소

#### 3.3.4 모델 4: ARIMA(1,1,1)

```
차분: ΔXₜ = Xₜ - Xₜ₋₁
모델: ΔXₜ = φ₁ΔXₜ₋₁ + εₜ + θ₁εₜ₋₁

파라미터 추정: CSS(조건부 제곱합) 최소화
예측: X̂ₜ₊ₕ = Xₜ + Σ(k=1→h) ΔX̂ₜ₊ₖ
```

#### 3.3.5 모델 5: ETS(A,A,A) - Holt-Winters 삼중 지수 평활

```
레벨:  lₜ = α(yₜ - sₜ₋ₘ) + (1-α)(lₜ₋₁ + bₜ₋₁)
추세:  bₜ = β(lₜ - lₜ₋₁) + (1-β)bₜ₋₁
계절:  sₜ = γ(yₜ - lₜ) + (1-γ)sₜ₋ₘ

예측:  ŷₜ₊ₕ = lₜ + h·bₜ + sₜ₊ₕ₋ₘ
```

여기서:
- `α, β, γ`: 평활 파라미터 (Grid Search로 최적화)
- `m`: 계절 주기 (12개월)

#### 3.3.6 앙상블 결합 및 거시경제 보정

```
P̂_ensemble = Σ(i=1→5) wᵢ × P̂ᵢ

거시경제 보정:
Δr = r_current - r_default    (기준금리 차이)
adj = Δr × (-0.005)           (금리 1%p ↑ → 가격 -0.5%)

P̂_final = P̂_ensemble × (1 + adj)
```

#### 3.3.7 신뢰도 점수 (Confidence Score)

```
C = C_data × 0.4 + C_fit × 0.3 + C_recency × 0.15 + C_volume × 0.15

C_data = min(100, 거래건수 × 3.3)
C_fit = R² × 100
C_recency = { 100: 최근3개월, 70: 최근6개월, 40: 최근12개월, 20: 기타 }
C_volume = min(100, 월평균거래량 × 20)
```

### 3.4 선행기술 조사

| 선행기술 | 출처 | 차이점 |
|---------|------|--------|
| 배성완·유정석 (2018) "머신러닝·시계열 분석을 이용한 부동산 가격지수 예측" | 한국주택학회 | SVM, RF, GBRT, DNN, LSTM vs ARIMA, VAR 비교 연구; 앙상블 결합이 아닌 개별 모델 성능 비교 |
| ARIMA 모형을 이용한 전국 아파트시장 가격 예측 | KCI 등재 논문 | ARIMA 단일 모델 사용, 앙상블 미적용 |
| AdaBoost-GRU 앙상블 (금융 시계열) | 한국데이터정보과학회지 | 금융 시계열 대상, 부동산 특화 거시경제 보정 없음 |
| 부동산 플래닛 AI 추정가 | 상용 서비스 | 헤도닉 가격 모형 기반, 수식 비공개 |

**차별성**: 본 기술은 (1) 성격이 상이한 5가지 모델(추세/회귀/모멘텀/시계열/계절성)을 앙상블하고, (2) 거시경제 지표(기준금리)를 실시간 보정하며, (3) 데이터 품질 기반 **신뢰도 전파 시스템**을 함께 산출하는 점이 기존 단일 모델 또는 동질적 앙상블과 구별됨.

### 3.5 파라미터 근거 및 검증 계획

> **현재 상태**: 5개 모델의 앙상블 가중치(wᵢ), 평균회귀 계수(-0.3), 모멘텀 감쇠율(0.15), 거시경제 보정 계수(-0.005), 신뢰도 산출 가중치(0.4/0.3/0.15/0.15)는 **통계적 직관 및 부동산 시장 일반론에 기반한 초기값**이다.

**검증 계획**:
1. **Rolling Window 백테스트**: `backtesting.ts`에 구현된 36개월 Rolling Window 방식으로 과거 MOLIT 실거래 데이터에 대한 예측 정확도(MAE, MAPE) 측정 — 이미 코드 존재, 실행 및 결과 문서화 필요
2. **모델별 성능 비교**: 5개 개별 모델 vs 앙상블의 MAE/MAPE를 비교하여 앙상블 효과 정량화
3. **거시경제 보정 계수 검증**: 한국은행 기준금리 변동 시점과 아파트 실거래가 변동률의 상관관계 분석으로 -0.005 계수의 타당성 확인
4. **신뢰도 점수 교정**: 예측 후 실제 가격과의 오차를 신뢰도 구간별로 집계하여, "신뢰도 80점 이상인 예측의 실제 오차율이 10% 이내"와 같은 정량적 기준 수립

## 4. 특허기술 C: V-Score 이질 데이터 통합 위험 평가 시스템

### 4.1 기술 명칭
**5종 이질 데이터 소스의 가중 결합 및 비선형 상호작용 규칙 기반 통합 부동산 위험도 평가 방법 (V-Score)**

### 4.2 기술 설명
등기 권리관계, 시세/전세가율, 계약서 안전성, 임대인 신용, 지역 위험도 등 **5종의 이질적 데이터 소스**를 단일 점수로 통합하면서, 소스 간 **6가지 비선형 상호작용 규칙**을 적용한다.

### 4.3 수식

#### 4.3.1 가중 결합

```
V_raw = Σ(k=1→5) wₖ × Sₖ

w = { registry: 0.30, price: 0.25, contract: 0.20, landlord: 0.15, region: 0.10 }
Σwₖ = 1.0
```

#### 4.3.2 개별 소스 점수 산출

**시세 점수 (S_price)**:
```
S_price = 0.7 × (100 - 전세가율) + 0.2 × max(0, 100 - 변동성×2) + 0.1 × 신뢰도
```

**임대인 점수 (S_landlord)**:
```
S_landlord = max(20, 100 - D_multi - D_corp - D_tax - D_credit)

D_multi = 20 (다주택), D_corp = 10 (법인), D_tax = 30 (세금체납), D_credit ≤ 20
```

**지역 점수 (S_region)**:
```
S_region = 0.6 × max(0, 100 - 사기율×10) + 0.4 × max(0, 100 - 경매율×10)
```

#### 4.3.3 비선형 상호작용 조정

```
V_adjusted = V_raw + Σ(j=1→6) δⱼ

δ₁ = -⌊(100 - S_reg)(100 - S_price) / 200⌋,   조건: S_reg < 50 ∧ S_price < 50
δ₂ = -8,                                         조건: S_reg < 40 ∧ S_contract < 40
δ₃ = -6,                                         조건: S_price < 50 ∧ S_region < 40
δ₄ = -5,                                         조건: S_landlord < 40 ∧ S_reg < 50
δ₅ = +3,                                         조건: S_contract ≥ 80 ∧ S_price ≥ 70  (경감)
δ₆ = -10,                                        조건: S_reg < 30 ∧ S_price < 30

V_score = clamp(V_adjusted, 0, 100)
```

### 4.5 파라미터 근거 및 검증 계획

## 5. 특허기술 D: 전세사기 위험도 Gradient Boosting 시뮬레이션 모델

### 5.1 기술 명칭
**15-피처 5-그룹 Gradient Boosting 시뮬레이션 및 SHAP 유사 기여도 기반 전세사기 위험 진단 방법**

### 5.2 기술 설명
15개 입력 피처를 5개 그룹(권리관계, 시세가격, 임대인, 건물지역, 계약조건)으로 구분하고, 규칙 기반 Gradient Boosting 시뮬레이션으로 전세사기 위험도를 산출하며, Leave-One-Out 방식의 **SHAP 유사 기여도**를 제공한다.

### 5.3 수식

#### 5.3.1 피처 가중 합산

```
F_raw = (Σ(i=1→15) vᵢ × wᵢ) / (Σ(i=1→15) wᵢ)
```

여기서:
- `vᵢ`: i번째 피처의 정규화된 위험값 (0~100)
- `wᵢ`: i번째 피처의 가중치

#### 5.3.2 피처 그룹 및 가중치

| 그룹 | 피처 | wᵢ | 위험값 계산 |
|-----|------|-----|-----------|
| 권리관계 (0.30) | 근저당비율 | 0.12 | min(100, ratio%) |
| | 압류존재 | 0.10 | min(100, 50 × count) |
| | 선순위채권비율 | 0.08 | min(100, ratio%) |
| 시세가격 (0.24) | 전세가율 | 0.14 | min(100, ratio%) |
| | 시세변동률 | 0.06 | min(100, volatility × 5) |
| | 공실률 | 0.04 | min(100, rate × 8) |
| 임대인 (0.18) | 다주택보유 | 0.06 | bool → 70 |
| | 법인임대 | 0.04 | bool → 50 |
| | 세금체납 | 0.08 | bool → 90 |
| 건물지역 (0.16) | 건축년수 | 0.03 | min(100, years × 2.5) |
| | 지역사기율 | 0.08 | min(100, rate × 20) |
| | 경매발생률 | 0.05 | min(100, rate × 25) |
| 계약조건 (0.12) | 계약서점수 | 0.06 | 100 - safetyScore |
| | 중개사등록 | 0.03 | 미등록 → 80 |
| | 보증보험 | 0.03 | 미가입 → 60 |

#### 5.3.3 비선형 고위험 증폭

```
n_high = count(vᵢ × wᵢ > 3)

       ┌ 1.2,  n_high ≥ 3
α =    │ 1.1,  n_high = 2
       └ 1.0,  otherwise

F_score = min(100, ⌊F_raw × α⌋)
```

#### 5.3.4 SHAP 유사 기여도 (Leave-One-Out)

```
Cᵢ = F(X) - F(X \ {xᵢ})

%Impactᵢ = |Cᵢ| / F(X) × 100,  if F(X) > 0
```

여기서:
- `F(X)`: 전체 피처로 계산한 점수
- `F(X \ {xᵢ})`: i번째 피처를 제외하고 계산한 점수

#### 5.3.5 유사 사례 매칭 (Haversine Distance)

```
d = 2R × arcsin(√(sin²(Δφ/2) + cos φ₁ × cos φ₂ × sin²(Δλ/2)))

R = 6371 km
similarity = max(0, 1 - d / d_max),  d_max = 5 km
```

| 선행기술 | 출처 | 차이점 |
|---------|------|--------|
| 국가AI전략위 전세사기 사전탐지 모델 | 국토부 (2025) | 300만건 계약정보 기반 ML, 금융지표 중심 60% 포착률 |

| 경기도 AI 거래 안전망 | 경기도 (2025) | 단계별 모니터링, 전용 피처 기여도 분석 없음 |

**차별성**: 본 기술은 (1) **해석 가능한 15-피처 규칙 기반** Gradient Boosting 시뮬레이션, (2) **SHAP 유사 Leave-One-Out 기여도**로 각 피처의 위험 기여를 정량화, (3) **Haversine 기반 유사 사례 매칭**으로 지리적 근접 사기 사례를 연결하는 점이 기존 블랙박스 ML 모델과 구별됨.

### 5.5 파라미터 근거 및 검증 계획

> **현재 상태**: 15개 피처 가중치(0.03~0.14), 위험값 변환 함수(min(100, ratio%), min(100, rate × 20) 등), 비선형 증폭 기준(n_high ≥ 3 → 1.2배), Haversine 매칭 반경(5km)은 **부동산 사기 유형 분석에 기반한 전문가 추정치**이다.

2. **SHAP 기여도 교차 검증**: Leave-One-Out 방식의 기여도와 Permutation Importance를 비교하여 기여도 산출 방식의 일관성 확인

## 6. 특허기술 E: 시계열 이상 패턴 탐지 엔진

### 6.1 기술 명칭
**등기부등본 시계열 데이터의 8-패턴 이상 탐지 및 Holt-CUSUM-Bollinger 복합 이상 감지 시스템**

### 6.2 기술 설명
등기부등본의 시간순 이벤트에서 전세사기 전조 패턴을 탐지하는 8가지 시계열 규칙과, 부동산 시세 시계열의 이상을 탐지하는 3중 통계 엔진(Holt's Smoothing + CUSUM + Adaptive Bollinger)을 결합한다.

### 6.3 수식

#### 6.3.1 등기부 8-패턴 이상 탐지

각 패턴 k에 대해:

```
Pattern_k = { type, condition, confidence_k, severity_k }

월간격 = |toMonths(date₁) - toMonths(date₂)|
toMonths(YYYY.MM.DD) = YYYY × 12 + MM
```

| 패턴 | 조건 | 신뢰도 | 심각도 |
|------|------|--------|--------|
| 급속소유권이전 | 60개월 내 ≥3회 이전 | min(1, n/5) | n≥4: critical |
| 압류 전 근저당 | 압류 6개월 이내 근저당 설정 | 0.9 | critical |
| 채권 가속 | 12개월 내 ≥2건 압류/가압류 | min(1, n/3) | n≥3: critical |
| 근저당 누적 | 3개월 내 ≥2건 근저당 설정 | 0.8 | high |
| 의심 말소 | 설정 후 6개월 내 말소 | gap≤3m: 0.9, ≤6m: 0.7 | high |
| 말소 후 즉시 매매 | 말소 1개월 내 소유권이전 | 0.8 | high |
| 동시 다건 말소 | 같은 날 ≥2건 근저당 말소 | 0.75 | high |
| 고액 말소 무이전 | 2억 이상 말소 후 6개월 내 매매 없음 | 0.6 | medium |

#### 6.3.2 시계열 종합 위험도

```
R_temporal = min(100, Σ(k) severity_weight(k) × confidence_k)

severity_weight = { critical: 30, high: 15, medium: 5 }

시계열이상점수 = min(100, 패턴수 × 평균신뢰도 × 25)
시계열감점 D_temporal = min(20, ⌊R_temporal × 0.2⌋)
```

#### 6.3.3 Holt's Double Exponential Smoothing

```
lₜ = α × yₜ + (1 - α)(lₜ₋₁ + bₜ₋₁)
bₜ = β × (lₜ - lₜ₋₁) + (1 - β)bₜ₋₁

ŷₜ₊ₕ = lₜ + h × bₜ

최적 파라미터: argmin(α,β) MSE = (1/(n-1)) Σ(yₜ - smoothedₜ)²
Grid Search: α ∈ [0.1, 0.9], β ∈ [0.01, 0.5]
```

#### 6.3.4 적응형 Bollinger Band

```
CV = σ(smoothed) / μ(smoothed)    (변동계수)
m_adaptive = m_base × (1 + CV)    (적응형 밴드 폭)

upper = smoothed + m_adaptive × σ
lower = smoothed - m_adaptive × σ

이상: yₜ > upper ∨ yₜ < lower
```

#### 6.3.5 CUSUM 변화점 탐지

```
C⁺ₜ = max(0, C⁺ₜ₋₁ + (yₜ - μ - k))
C⁻ₜ = max(0, C⁻ₜ₋₁ - (yₜ - μ + k))

변화점: C⁺ₜ > h ∨ C⁻ₜ > h

h = 임계값 (3~5σ), k = 0.5σ
```

#### 6.3.6 Robust Z-Score (MAD 기반)

```
MAD = median(|yᵢ - median(y)|)
z_robust = (y - median) / (1.4826 × MAD)

이상: |z_robust| > 3 (고위험), |z_robust| > 2 (중위험)
```

### 6.4 선행기술 조사

| 선행기술 | 출처 | 차이점 |
|---------|------|--------|
| CoreLogic US20140025548A1 | 부동산 거래 이상 탐지 | 거래 금액/빈도 패턴, 등기부 시계열 이벤트 패턴 아님 |
| CUSUM 신호 이상 탐지 (IEEE) | 신호 처리 분야 | 범용 신호 처리, 부동산 등기부 특화 패턴 없음 |
| Bollinger Band 기반 태양광 모니터링 | MDPI Energies | 에너지 분야, 부동산 시세 적용 없음 |
| HomeBuyer 온라인 트래픽 이상 탐지 | SARIMA + Bollinger | 방문자 트래픽 대상, 시세/등기부 대상 아님 |

**차별성**: 본 기술은 (1) **부동산 등기부 특화 8-패턴 이상 탐지 규칙**, (2) **Holt + CUSUM + Adaptive Bollinger 3중 통계 엔진**의 결합, (3) 등기부 이벤트와 시세 시계열을 **교차 분석**하는 점이 기존 범용 이상 탐지 기술과 구별됨. 특히 "의심스러운 근저당 말소" "동시 다건 말소" 등의 패턴은 한국 부동산 등기 제도에 특화된 도메인 지식 기반 규칙임.

### 6.5 파라미터 근거 및 검증 계획

> **현재 상태**: 8-패턴의 기간 조건(60개월, 6개월, 12개월, 3개월 등), 신뢰도 값(0.6~0.9), Holt 파라미터 탐색 범위, CUSUM 임계값(3~5sigma), Adaptive Bollinger 밴드 폭 계수는 **통계학 교과서 권장값 + 부동산 거래 주기의 경험적 추정**에 기반한다.

## 7. 특허기술 F: Thompson Sampling 기반 적응형 가중치 자동 튜닝

### 7.1 기술 명칭
**Thompson Sampling 기반 베이지안 최적화 및 Isotonic Regression 확률 보정을 통한 부동산 위험 평가 가중치 자동 튜닝 시스템**

### 7.2 기술 설명
위험 평가 모델의 가중치를 사후 데이터(실제 사기 발생 여부)를 기반으로 **자동 최적화**하며, 확률 예측의 보정(calibration)을 Isotonic Regression으로 수행한다.

### 7.3 수식

#### 7.3.1 Thompson Sampling (Beta Distribution)

```
각 가중치 후보 i에 대해:
αᵢ = 성공횟수 + 1
βᵢ = 실패횟수 + 1

θᵢ ~ Beta(αᵢ, βᵢ)

선택: i* = argmax(i) θᵢ
```

#### 7.3.2 Gamma Distribution Sampling (Marsaglia & Tsang)

```
shape < 1: Gamma(shape+1) × U^(1/shape)
shape ≥ 1: Marsaglia rejection method

Beta(α,β) = Gamma(α) / (Gamma(α) + Gamma(β))
```

#### 7.3.3 성능 메트릭

**Brier Score** (확률 예측 정확도):
```
BS = (1/N) Σ(pᵢ - oᵢ)²    (0 = 완벽, 1 = 최악)
```

**Expected Calibration Error (ECE)**:
```
ECE = Σ(b=1→B) (nᵦ/N) × |avg(pᵦ) - avg(oᵦ)|

B = 10 (빈 개수)
```

**Log Loss**:
```
LL = -(1/N) Σ(yᵢ log(pᵢ) + (1-yᵢ) log(1-pᵢ))
```

#### 7.3.4 Isotonic Regression (PAVA 알고리즘)

```
입력: (pᵢ, oᵢ) 쌍을 pᵢ 기준 정렬

반복:
  단조성 위반 탐지: cal[i] > cal[i+1]
  위반 시 병합: cal_merged = (cal[i]×w[i] + cal[i+1]×w[i+1]) / (w[i] + w[i+1])
  이전 블록 재검사

출력: 단조 증가하는 보정된 확률 함수
```

### 7.4 선행기술 조사

| 선행기술 | 출처 | 차이점 |
|---------|------|--------|
| Parallelised BO via Thompson Sampling | PMLR (Kandasamy 2018) | 범용 하이퍼파라미터 튜닝, 부동산 위험 가중치 특화 아님 |
| Fast Thompson Sampling for BO | arXiv 2411.17071 | GP 기반, 규칙 가중치 튜닝 아님 |
| Federated BO via Thompson Sampling | arXiv 2010.10154 | 분산 학습, 단일 시스템 적응형 튜닝 아님 |

**차별성**: 본 기술은 Thompson Sampling을 **부동산 위험 평가 가중치 자동 튜닝**에 특화 적용하고, **Isotonic Regression(PAVA)으로 확률 보정**을 결합한 점이 기존 범용 베이지안 최적화와 구별됨. 특히 Brier Score + ECE + LogLoss 3가지 메트릭을 종합하여 가중치 후보를 평가하는 **다기준 최적화 프레임워크**가 핵심.

### 7.5 파라미터 근거 및 검증 계획

> **현재 상태**: Thompson Sampling, PAVA 알고리즘 자체는 **학술적으로 검증된 표준 방법론**이다. 다만 이를 부동산 위험 평가 가중치 튜닝에 적용한 사례는 없으며, 튜닝 대상인 상위 모델(A~E)의 파라미터가 실증 검증되지 않은 초기값이므로, **튜닝의 기준(ground truth) 자체의 품질이 선행 과제**이다.

**검증 계획**:
1. **Ground Truth 구축이 선행 필수**: 특허기술 A~E의 파라미터를 실증 데이터로 1차 캘리브레이션한 후, 본 시스템으로 2차 자동 튜닝 적용
2. **수렴성 검증**: Thompson Sampling이 부동산 위험 가중치 공간에서 합리적 시간 내 수렴하는지 시뮬레이션 실행
3. **PAVA 보정 효과 측정**: 보정 전/후 Calibration Plot 비교로 Isotonic Regression 적용 효과 정량화

## 8. 특허기술 G: 4단계 등기 데이터 자동 검증 엔진

### 8.1 기술 명칭
**등기부등본 파싱 데이터의 4단계 자동 검증 및 데이터 신뢰도 점수 산출 시스템**

### 8.2 기술 설명
AI가 파싱한 등기부등본 데이터의 정확성을 **포맷 → 산술 → 문맥 → 크로스체크** 4단계로 자동 검증하여 데이터 신뢰도 점수를 산출한다.

### 8.3 수식

#### 8.3.1 4단계 검증 체계

**Stage 1: 포맷 검증**
```
날짜: YYYY.MM.DD, 1900 ≤ Y ≤ 2035, 1 ≤ M ≤ 12, 1 ≤ D ≤ 31
금액: 1,000,000 ≤ amount ≤ 50,000,000,000
순위: rank > 0 ∧ unique(ranks)
권리자: 2 ≤ len(name) ≤ 30
```

**Stage 2: 산술 검증**
```
검증 1: Σ(활성 근저당액) = summary.totalMortgageAmount
검증 2: Σ(활성 전세권액) = summary.totalJeonseAmount
검증 3: 총채권액 = 근저당합계 + 전세권합계
검증 4: |재계산 근저당비율 - 기존 비율| ≤ 0.1%
검증 5: count(활성 갑구) = summary.activeGapguEntries
```

**Stage 3: 문맥 검증**
```
시간순: ∀i < j, date(entry_i) ≤ date(entry_j)
말소 원본: 말소 참조 번호에 해당하는 원본 항목 존재
소유권 체인: count(활성 소유권보존/이전) ≥ 1
갑구 1번: purpose ∈ {"소유권보존", "소유권이전"}
```

**Stage 4: 크로스체크**
```
∀flag ∈ {hasSeizure, hasTrust, hasAuction, ...}:
  flag == exists(해당 활성 항목)

추정가격: 50,000,000 ≤ P_est ≤ 50,000,000,000
리스크 스코어: totalScore = max(0, 100 - Σ(deductions))
```

#### 8.3.2 신뢰도 점수

```
N_total = 전체 검증 항목 수
N_pass = N_total - N_error - N_warning

신뢰도 = max(0, N_pass / N_total) × 100

isValid = (N_error == 0)
```

### 8.4 선행기술 조사

등기부등본 파싱 데이터의 **자동 다단계 검증 시스템**에 대한 직접적 선행 특허는 발견되지 않음. 기존 부동산 AI 서비스는 파싱 결과를 그대로 사용하며, 파싱 데이터 자체의 정합성을 체계적으로 검증하는 시스템은 공개된 바 없음.

**차별성**: AI 파싱 결과의 **자기 검증(Self-Validation)** 개념이 핵심. 4단계 파이프라인으로 포맷·산술·문맥·크로스체크를 순차 수행하고, 데이터 신뢰도 점수를 산출하여 후속 분석의 신뢰성을 보장함.

## 9. 특허기술 H: 부동산 세금 통합 시뮬레이션 엔진

### 9.1 기술 명칭
**취득세·보유세·양도세 통합 시뮬레이션 및 연도별 세법 변경 자동 적용 부동산 세금 계산 시스템**

### 9.2 기술 설명
취득세(누진세율+다주택중과+생애최초감면), 재산세(공시가격 구간), 종합부동산세(공정시장가액비율), 양도소득세(장기보유특별공제+중과)를 통합 계산하며, 연도별 세법 변경을 자동 적용한다.

### 9.3 수식

#### 9.3.1 취득세

**1주택 누진세율**:
```
         ┌ 1%,                                    P ≤ 6억
rate₁ =  │ 1% + (P - 6억) / 3억 × 2%,            6억 < P ≤ 9억
         └ 3%,                                    P > 9억

T_acq = P × rate
T_edu = T_acq × 0.1                              (지방교육세)
T_rural = P × 0.002  (rate > 2% 시에만)           (농어촌특별세)
T_total_acq = T_acq + T_edu + T_rural
```

**생애최초 감면**:
```
조건: isFirstHome ∧ houseCount = 1 ∧ P ≤ 12억

         ┌ 0,                     P ≤ 6억
T_acq =  │ max(0, P×1% - 200만),  6억 < P ≤ 12억
         └ 정상세율,               P > 12억
```

**다주택 중과 (multiHomeHeavyTaxEnabled)**:
```
2주택 조정지역: 8%
3주택 조정지역: 12%
4주택 이상:    12%
```

#### 9.3.2 재산세 (공시가격 기준)

```
         ┌ V × 0.1%,                          V ≤ 6천만
         │ 6만 + (V - 6천만) × 0.15%,          6천만 < V ≤ 1.5억
T_prop = │ 19.5만 + (V - 1.5억) × 0.25%,      1.5억 < V ≤ 3억
         └ 57만 + (V - 3억) × 0.4%,           V > 3억
```

#### 9.3.3 종합부동산세

```
과세표준 = V_assessed × fairMarketRatio - 공제액

공제액 = { 12억: 1세대1주택, 9억: 그 외 }

일반세율 (≤2주택):
         ┌ B × 0.5%,                              B ≤ 3억
         │ 150만 + (B-3억) × 0.7%,                3억 < B ≤ 6억
T_comp = │ 360만 + (B-6억) × 1.0%,                6억 < B ≤ 12억
         │ 960만 + (B-12억) × 1.3%,               12억 < B ≤ 25억
         │ 2,650만 + (B-25억) × 1.5%,             25억 < B ≤ 50억
         └ 6,400만 + (B-50억) × 2.0%,             B > 50억
```

#### 9.3.4 양도소득세

```
양도차익 = P_sell - P_buy
필요경비 = P_buy × 3%
과세대상 = max(0, 양도차익 - 필요경비)

1세대1주택 비과세: holdingYears ≥ 2 ∧ P_sell ≤ 12억 → T = 0

12억 초과분 과세:
  taxableRatio = (P_sell - 12억) / P_sell
  조정차익 = 과세대상 × taxableRatio

장기보유특별공제:
  보유공제 = min(holdingYears × 4%, 40%)
  거주공제 = min(residenceYears × 4%, 40%)
  총공제 = min(보유공제 + 거주공제, 80%)

최종차익 = 조정차익 × (1 - 총공제)
```

**누진세율**:
```
         ┌ 6%,                                    과세소득 ≤ 1,400만
         │ 15% - 126만,                           ≤ 5,000만
         │ 24% - 576만,                           ≤ 8,800만
T_trans = │ 35% - 1,544만,                        ≤ 1.5억
         │ 38% - 1,994만,                         ≤ 3억
         │ 40% - 2,594만,                         ≤ 5억
         │ 42% - 3,594만,                         ≤ 10억
         └ 45% - 6,594만,                         > 10억
```

### 9.4 선행기술 조사

부동산 세금 계산기는 다수 존재하나, **취득세+보유세+양도세를 통합 시뮬레이션하면서 연도별 세법 변경(중과 유예/복원)을 자동 적용**하는 시스템은 일반적이지 않음. 대부분의 서비스는 개별 세목별 계산기를 제공함.

## 10. 선행기술 대비 차별성 종합 분석

### 10.1 VESTRA 고유 기술 매트릭스

| 특허 | 핵심 차별 요소 | 선행기술 존재 여부 | 특허 가능성 |
|-----|--------------|-----------------|------------|
| **A** | 13-팩터 + 6-상호작용 비선형 증폭 | 유사 개념 있으나 비선형 증폭 결합 없음 | **높음** |
| **B** | 5-모델 이질 앙상블 + 거시경제 보정 + 신뢰도 전파 | 개별 모델 연구 다수, 5-모델 앙상블 + 신뢰도 결합 없음 | **높음** |
| **C** | 5-소스 V-Score + 6-상호작용 (증폭+경감) | 다소스 통합 개념 있으나 비선형 상호작용 규칙 기반은 없음 | **매우 높음** |
| **D** | 15-피처 SHAP 유사 기여도 + Haversine 사례 매칭 | ML 기반 전세사기 탐지 연구 있으나 해석가능 규칙+SHAP 결합 없음 | **높음** |
| **E** | 8-패턴 등기부 시계열 + 3중 통계 엔진 | CUSUM/Bollinger 개별 사용 있으나 등기부 특화 결합 없음 | **매우 높음** |
| **F** | Thompson Sampling + PAVA 확률 보정 for 부동산 | 범용 BO 연구 다수, 부동산 위험 가중치 특화 적용 없음 | **높음** |
| **G** | 4단계 파싱 데이터 자기 검증 | 직접적 선행기술 미발견 | **매우 높음** |
| **H** | 3세목 통합 + 연도별 세법 자동 적용 | 개별 세금 계산기 다수, 통합 시뮬레이션 시스템은 희소 | **중간** |

### 10.2 종합 의견

VESTRA의 핵심 경쟁력은 **개별 알고리즘의 독창성**보다 **한국 부동산 도메인에 특화된 복합 시스템 통합**에 있다. 특히:

1. **비선형 상호작용 규칙**: 단순 가중합이 아닌 팩터/소스 간 증폭·경감 메커니즘
2. **해석 가능성**: 블랙박스 ML이 아닌 규칙 기반 + SHAP 유사 기여도
3. **자기 검증**: AI 파싱 → 4단계 자동 검증 → 위험도 산출의 파이프라인
4. **도메인 특화**: 한국 등기부등본 제도, 전세 제도, 세법에 특화된 규칙 세트

이러한 요소들의 **유기적 결합**이 가장 강력한 특허 청구 포인트이며, 개별 기술보다 **시스템 전체로서의 방법 특허**를 출원하는 것이 전략적으로 유리할 것으로 판단됨.

### 10.3 파라미터 실증 검증 로드맵

본 문서에 기술된 자체 설계 파라미터(가중치, 감점 수치, 증폭계수, 기간 조건 등)는 **도메인 전문가 휴리스틱에 기반한 초기값**이며, 특허 출원 및 서비스 신뢰성 확보를 위해 아래 로드맵에 따른 실증 검증이 필요하다.

#### Phase 1: 데이터 수집 (1~2개월)

| 데이터 | 출처 | 용도 |
|-------|------|------|
| 전세보증사고 내역 | HUG(주택도시보증공사) | 사고/안전 물건 라벨링 |
| 경매 개시 결정 물건 | 법원 경매정보 | 고위험 물건 식별 |
| 전세사기 판례 | 대법원 판례검색 | 사기 패턴 실증 |
| MOLIT 실거래 36개월 | 국토교통부 | 시세 예측 백테스트 |
| 등기부등본 이력 | 인터넷등기소 | 시계열 패턴 검증 |

#### Phase 2: 후향적 검증 (2~3개월)
1. 사고 물건 등기부를 모델에 입력 → 사고 전 시점 위험 등급 측정 (목표: 민감도 80% 이상)
2. 안전 물건 등기부를 모델에 입력 → 오경보율(False Positive) 측정 (목표: 20% 이하)
3. 시세 예측 Rolling Window 백테스트 → MAE/MAPE 측정

#### Phase 3: 파라미터 캘리브레이션 (1~2개월)
1. 특허기술 F(Thompson Sampling)를 활용하여 A~E 모델의 가중치 자동 최적화
2. 전문가 패널(법무사, 감정평가사, 부동산 변호사) 자문으로 정성적 타당성 검증
3. 최적화된 파라미터를 코드에 반영하고 주석으로 근거 명시

#### Phase 4: 전향적 검증 및 A/B 테스트 (지속)
1. 실서비스에서 초기값 vs 캘리브레이션 값의 예측 정확도 비교
2. 사용자 피드백("이 분석이 도움이 되었나요?") 수집 및 반영
3. 분기별 파라미터 리뷰 및 재캘리브레이션

> **특허 출원 시점**: Phase 2 완료 후 "민감도 80% 이상, 오경보율 20% 이하"의 실증 결과가 확보된 시점에서 출원하는 것이 심사관 대응에 유리함. 현재 시스템 구조(방법 특허)는 Phase 1 전에도 출원 가능하나, 구체적 수치의 기술적 효과 입증이 약할 수 있음.
